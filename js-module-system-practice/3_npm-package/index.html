외부 라이브러리 사용 실습

npm으로 외부 패키지를 설치하고, import로 가져와서 사용하는 실습입니다.
이 실습은 터미널(명령어)에서 진행합니다. 각 단계의 안내를 따라 진행하세요.

사전 준비: Node.js 설치 확인

터미널을 열고 아래 명령어를 입력하여 Node.js가 설치되어 있는지 확인합니다.

node --version npm --version
버전 번호(예: v20.x.x, 10.x.x)가 나오면 설치된 것입니다!
설치가 안 되어 있다면 https://nodejs.org 에서 LTS 버전을 설치하세요.

파일 구조

이번 실습에서 작업할 파일들입니다.

3_npm-package/ ├── index.html ← 지금 보고 있는 파일 (안내용) ├── package.json ← 프로젝트 설정 파일 (npm init으로 생성) ├── main.js ← 메인 실행 파일 ├── date-utils.js ← 날짜 유틸리티 (dayjs 사용) └── fake-data.js ← 가짜 데이터 생성 (@faker-js/faker 사용)
이 실습은 브라우저가 아닌 터미널에서 실행합니다!
터미널에서 node main.js 명령어로 결과를 확인하세요.

섹션 1: 프로젝트 초기화

터미널에서 이 폴더(3_npm-package)로 이동한 뒤 아래 명령어를 실행하세요.

# 이 폴더(3_npm-package)에서 실행 npm init -y

package.json 파일이 생성됩니다. 생성된 파일을 열고 아래 내용을 추가하세요.

package.json에 "type": "module"을 추가해야 import/export 문법을 사용할 수 있어요!

수정 전: "main": "index.js",
수정 후: "main": "index.js", 아래에 "type": "module", 추가

수정 후 package.json은 이렇게 되어야 합니다 (핵심 부분만):

{ "name": "3_npm-package", "version": "1.0.0", "main": "index.js", "type": "module", ... }

섹션 2: 패키지 설치

터미널에서 아래 명령어로 2개의 패키지를 설치하세요.

# dayjs: 날짜를 쉽게 다루는 패키지 # @faker-js/faker: 가짜 데이터를 생성하는 패키지 npm install dayjs @faker-js/faker

설치가 완료되면 아래 변화를 확인해보세요.

확인할 것 3가지:
1. node_modules 폴더가 생겼나요? (패키지가 실제로 저장되는 곳)
2. package-lock.json 파일이 생겼나요? (정확한 버전 기록)
3. package.json의 dependencies에 dayjs, @faker-js/faker가 추가되었나요?
파일/폴더 역할 Git에 올리나요?
node_modules/ 패키지 실제 코드가 저장되는 곳 올리지 않음 (용량이 큼)
package.json 프로젝트 설정 + 설치된 패키지 목록 올림
package-lock.json 설치된 패키지의 정확한 버전 기록 올림

섹션 3: dayjs 사용하기 — date-utils.js

date-utils.js를 열고 TODO를 완성하세요.

설치한 패키지는 경로 없이 이름만 써서 import 합니다!
import dayjs from 'dayjs'; (경로 ./ 없음!)

dayjs 주요 사용법:
dayjs() — 현재 날짜/시간 객체 생성
.format('YYYY-MM-DD') — 원하는 형식으로 날짜 출력
.add(숫자, 'day') — 날짜 더하기
.subtract(숫자, 'day') — 날짜 빼기

완성 후 기대하는 출력 결과:

// 실행: node main.js === dayjs 테스트 === 오늘 날짜: 2026년 04월 15일 ← 실행한 날에 따라 다름 100일 뒤: 2026-07-24 ← 실행한 날에 따라 다름 7일 전: 2026-04-08 ← 실행한 날에 따라 다름

섹션 4: @faker-js/faker 사용하기 — fake-data.js

fake-data.js를 열고 TODO를 완성하세요.

@faker-js/faker 패키지에서 faker 객체를 가져옵니다. (named export)
import { faker } from '@faker-js/faker';

faker 주요 사용법:
faker.person.fullName() — 가짜 이름 생성
faker.internet.email() — 가짜 이메일 생성
faker.commerce.productName() — 가짜 상품명 생성
faker.commerce.price() — 가짜 가격 생성

완성 후 기대하는 출력 결과:

// 실행: node main.js === faker 테스트 === --- 가짜 유저 1명 --- 이름: John Smith ← 매번 다른 값 이메일: john.smith@example.com ← 매번 다른 값 전화번호: +1-555-123-4567 ← 매번 다른 값 아바타: https://avatars... ← 매번 다른 값 --- 가짜 상품 1개 --- 상품명: Handmade Wooden Chair ← 매번 다른 값 가격: 45000원 ← 매번 다른 값 카테고리: Electronics ← 매번 다른 값 설명: A beautiful product... ← 매번 다른 값

섹션 5: 모듈 조합하기 — main.js

main.js를 열고 TODO를 완성하세요.

직접 만든 모듈(date-utils.js, fake-data.js)을 함께 사용합니다.

import 비교:
우리가 만든 모듈: import { getToday } from './date-utils.js'; — 경로 ./ 필요!
설치한 패키지: import dayjs from 'dayjs'; — 경로 없이 이름만!

모든 TODO를 완성한 뒤 터미널에서 실행하세요:

node main.js

참고: npm 명령어 치트시트

명령어 설명 예시
npm init -y 프로젝트 초기화 (package.json 생성)
npm install 패키지 패키지 설치 npm install dayjs
npm i 패키지 패키지 설치 (약어) npm i dayjs @faker-js/faker
npm install package.json 기반 전체 설치
npm uninstall 패키지 패키지 제거 npm uninstall dayjs
node 파일이름 JS 파일 실행 node main.js
import 구분법:
우리가 만든 모듈 → import { add } from './utils.js'; (경로 ./ 필요)
설치한 패키지 → import dayjs from 'dayjs'; (이름만!)
