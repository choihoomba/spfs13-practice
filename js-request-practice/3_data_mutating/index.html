데이터 생성/수정/삭제 (POST, PUT, PATCH, DELETE)

fetch()의 두 번째 인자에 옵션 객체를 전달하면 GET 이외의 HTTP 메서드를 사용할 수 있습니다.
이 실습에서는 JSONPlaceholder API로 게시물을 생성, 수정, 삭제하는 CRUD 작업을 연습합니다.

📦 fetch 옵션 구조

GET 이외의 요청에는 method, headers, body를 설정해야 합니다.

method → HTTP 메서드 ('POST', 'PUT', 'PATCH', 'DELETE')
headers → { 'Content-Type': 'application/json' } (JSON 전송 시 필수)
body → JSON.stringify(데이터) (전송할 데이터를 JSON 문자열로)

기본 URL: https://jsonplaceholder.typicode.com
⚠️ JSONPlaceholder는 테스트용 API입니다. POST/PUT/DELETE 요청을 보내면 응답은 오지만 실제로 서버 데이터가 변경되지는 않습니다. 실제 서버와 동일한 형태의 응답을 돌려주므로 연습용으로 적합합니다.

💡 PUT vs PATCH
PUT: 전체 데이터를 교체 (보내지 않은 필드는 사라질 수 있음)
PATCH: 보낸 필드만 수정 (나머지 필드는 유지)
